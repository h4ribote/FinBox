# sim/ — パラメーター妥当性検証スクリプト

`doc/` の設計仕様で定めた既定パラメーター (賃金・物価・ニーズ減衰/回復・税率・金利・減価・戦闘式・生産レシピ・通貨供給) が経済的に妥当か (生存・採算・整合・決定論) を独立に検証するための、純 Python のスタンドアロン・スクリプト群である。これは FinBox の本体エンジンではなく、設計値を裏付けるための較正/検証ツールである。

実行: `python sim/<script>.py` (Python 3.12+、標準ライブラリのみ。numpy 不要)。すべての金額は minor 単位 (`money.minor_unit = 1000`、[doc/16](../doc/16-configuration-and-initialization.md) §16.3.1)。

## スクリプト

| script | 検証内容 | 主な結論 |
| --- | --- | --- |
| `household.py` | 労働者の生存ループ (労働→賃金→消費→ニーズ) を 480 ターン(10年) 動的シミュレート。doc 05 と doc 16 のニーズ減衰の双方を比較 | どちらの減衰でも生存可・餓死なし。賃金が中核消費を上回る |
| `checks.py` | 戦闘 `P_capture` 曲線と x1000 固定小数、クーポン(四半期 floor)、減価(年率)、複利 `g_turn`、YTM 二分法、通貨供給 (M0 ミント式の整合) | 戦闘・金融式は妥当。YTM は固定64反復で十分。M0 ミント式の過少 (外国MM・企業の取りこぼし) を検出 |
| `supply.py` | レシピ(doc 10)からの生産コスト積み上げ。最終財が参照価格で採算が取れるか | `recipe_yield_scale=1` では全最終財赤字(food 原価≈586,000)。`≈10` で採算成立 |
| `calibrate_prices.py` | `recipe_yield_scale=10`・マークアップ約1.4 で全生産段の単段マージンが正になる自己整合な参照価格表を固定点反復で算定 | doc/16 §16.15.1 の参照価格表の根拠 (最小マージン約28%) |
| `verify_prices.py` | doc/16 §16.15.1 の参照価格で各レシピの単段 P&L マージンを検証 | 全生産段が正マージンであることを確認 |

## 検証が導いた設計修正

- `production.recipe_yield_scale` の既定を 1 → **10** (供給連鎖の採算性。[doc/16](../doc/16-configuration-and-initialization.md) §16.6)。
- `M0_initial` ミント式を「自国通貨の genesis 配賦総額」に修正 (外国 MM 保有分・genesis 企業分を計上。§16.7.3)。
- genesis 参照価格表をコスト積み上げで自己整合に較正 (§16.15.1)。
- ニーズ減衰・餓死閾値を doc 05 (正準) と doc 16 §16.5 で一致。
- 戦闘係数・固定小数、厚生関数の正規化係数、YTM 固定反復、MM/報酬係数を確定 (§16.15)。

これらは設計値の検証用であり、本体エンジン実装時には同一の不変条件 (保存則・決定論・整数規律) を回帰テストとして組み込む ([doc/02](../doc/02-architecture.md)・[doc/16](../doc/16-configuration-and-initialization.md) §16.11)。
